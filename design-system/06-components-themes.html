Themes kit

Five opinionated looks a photographer can apply to their public portfolio with a single click. Themes share primitives (spacing, radii, tile logic), but each owns its typography, color, and gallery logic. Swapping a theme is a data-theme attribute on the site root — nothing else.

6.1 How it works

One attribute switches everything.

Every photographer site root has data-theme="elegante|moderno|minimalista|escuro|galeria". All colors, fonts and accents resolve through --t-* tokens in tokens.css. Components check a few theme-name-specific selectors for layout variations (masonry vs. grid, different gallery aspect ratios). Nothing else is themed.

Hard rule: no theme-specific JavaScript. Themes must work with a style-sheet swap alone. Animations and interactions are shared across all 5.

6.8 Building a new theme

What a new theme must define.

We keep the five above as the product offer. To prototype a sixth for A/B testing, add a new [data-theme='name'] block in tokens.css defining at minimum:

Required tokens
--t-bg, --t-bg-alt, --t-ink, --t-ink-muted, --t-primary, --t-accent, --t-border, --t-font-body, --t-font-display.
Optional tokens
--t-gradient, --t-accent-rgb.
Permitted overrides
Gallery grid (columns/aspect/gap), button radius + casing, nav/hero padding, headline weight/style.
Forbidden overrides
Component APIs, JS, icon library, admin/marketing surfaces.
